11 ## Requirements / Assumptions
12 Project requires and evironment with Ubuntu 18.04, ROS Melodic

15 ### Step 1. Clone repository and build the package
16 ```bash
17 git clone --recursive https://github.com/kavyadevd/beginner_tutorials.git
18 cp <repository_path> <catkin_workspace_path/src/>
19 cd <catkin_workspace>
20 source ./devel/setup.bash
21 catkin_make
22 ```
23 
24 ### Step 2. Make sure that a roscore is up and running
25 ```bash
26 roscore
27 ```
28 ### Step 3. In new terminal run publisher node
29 ```bash
30 rosrun beginner_tutorials talker
31 ```
32 ### Step 4. In new terminal run subscriber node
33 ```bash
34 rosrun beginner_tutorials listener
35 ```

98 ## ROSbag
99 rosbag is a set of tools for recording from and playing back to ROS topics. It is intended to be high performance and avoids deserialization and reserialization of the messages. [Read more](http://wiki.ros.org/rosbag)
100 
101 To record a bagfile run following commands:
102 Terminal 1
103 ```bash
104 rosrun beginner_tutorials talker
105 ```
106 Terminal 2
107 ```bash
108 mkdir ~/bagfiles
109 cd ~/bagfiles
110 rosbag record -a
111 ```
112 This will record till you stop it by typing Ctrl+C
113 
114 Check recorded bag file info using the command:
115 ```bash
116 rosbag info <bag-file-name>
117 ```
118 
119 To check the validity of bag file we will start listener node and check if the recorded messages are heard by listener.
120 
121 Terminal 1:
122 ```bash
123 rosrun beginner_tutorials listener
124 ```
125 
126 Terminal 2:
127 ```bash
128 rosbag play <rosbag-file-path>/<rosbag-file>.bag
129 ```

139 - CppCheckEclipse
140 
141  To install and run cppcheck in Eclipse
142 
143  1. In Eclipse, go to Window -> Preferences -> C/C++ -> cppcheclipse.
144  Set cppcheck binary path to "/usr/bin/cppcheck".
145 
146  2. To run CPPCheck on a project, right-click on the project name in the Project Explorer
147  and choose cppcheck -> Run cppcheck.
148 
149  3. To run on terminal
150  ```bash
151  cppcheck --enable=all --std=c++11 -I include/ --suppress=missingIncludeSystem $( find . -name *.cpp -or -name *.h | grep -vE -e "^./build/" -e "^./vendor/") > Results/cppcheckoutput.xml
152  ```
153  Results are present at Results/cppcheckoutput.xml
154 
155 - Cpplint
156  1. To run cpplint on terminal
157  ```bash
158  cpplint $( find . -name *.cpp | grep -vE -e "^./build/" -e "^./vendor/") $( find . -name *.hpp | grep -vE -e "^./build/" -e "^./vendor/") > Results/cpplintoutput.txt
159  ```
160  Results are present at Results/cpplintoutput.xml
